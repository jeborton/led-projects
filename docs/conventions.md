# Conventions

## Folder naming
- Use lowercase
- Use hyphens instead of spaces
- Examples:
  - `float-lights`
  - `top-hat`
  - `captains-hat`

## File naming
Use descriptive names that include project, purpose, and version.

Examples:
- `koss-sound-heavy-2A.js`
- `koss-quiet-waves-v2.js`
- `top-hat-comet-base-v1.py`
- `captains-hat-button-control-v1.ino`

## Code placement
- Trusted current files go in `stable/`
- Active work goes in `experimental/`
- Older retired versions go in `archived/`

## Documentation
Each project should include:
- `README.md`
- hardware notes
- wiring notes
- current base file or pattern reference
