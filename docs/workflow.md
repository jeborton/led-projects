# Workflow

## When creating or updating files
1. Put new work in `experimental/`
2. Promote it to `stable/` once it is trusted
3. Move replaced stable files to `archived/` if you want to preserve them

## When working with ChatGPT
Provide:
- the project name
- the platform (`pixelblaze`, `circuitpython`, or `arduino`)
- the current base file
- the goal of the change

Example:
"Update `float-lights/koss/pixelblaze/stable/koss-quiet-waves-v2.js` to make comet tails longer without reducing FPS."
